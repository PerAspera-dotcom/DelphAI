# Impeccable — DelphAI

Persistent design context for this repository. Update the **Design Context** section when positioning or audience changes.

## Design Context

### Users

DelphAI serves **reflective adults** who want a **philosophical thinking partner**—not quick answers or entertainment. They are often alone, focused, and using the product to explore ethics, abstract ideas, and personal sense-making. The **age gate** signals a serious, compliance-aware product stance. The job to be done is **structured reflection and perspective** in a calm, readable chat column, on desktop or mobile, locally or on Vercel.

### Brand Personality

**Voice:** clear, respectful, intellectually warm—never preachy or gimmicky. **Three words:** *thoughtful, grounded, luminous* (aligned with “Think deeper. See further.”). **Emotional goals:** confidence while thinking, calm focus, and trust in the tool as a quiet companion rather than a loud assistant.

### Aesthetic Direction

**Visual tone:** minimal, reading-first, **literary-adjacent**—serif accents (quotes, wordmark) paired with a humanist sans for UI and inputs. **Color:** neutral surfaces with CSS variables; **sage green** (`#4a6741`) for primary intent (CTAs, logo accent, quote emphasis); **muted blue-gray** (`#5a7a9a`) as a secondary system accent in tokens. **Theme:** **light and dark** following `prefers-color-scheme` (see `app/globals.css`). **Layout:** narrow column (~800px app, tighter age gate)—editorial, not dashboard-dense.

**Feel references (inferred):** quiet reading apps and editorial sites—generous type, restraint, no chrome overload.

**Anti-references:** neon “AI slop” palettes, gamification, social-chat clutter, aggressive motion, or stock “assistant” tropes that undermine contemplation.

### Design Principles

1. **Reading first** — Prioritize typography, measure, and whitespace so long replies and quotes remain easy to parse.
2. **Quiet confidence** — Use color and motion sparingly; let the content carry weight; keep transitions subtle.
3. **Token coherence** — Prefer extending `globals.css` variables and patterns in `page.module.css` over scattering one-off hex values.
4. **System-aware presentation** — Respect OS light/dark; for any motion beyond minimal hover/transition, honor **`prefers-reduced-motion`** (baseline today is light motion only).
5. **Accessible baseline** — Semantic structure, visible focus states, and contrast that holds in both themes; treat **WCAG 2.1 AA** as the default bar unless the product explicitly targets stricter levels.

## Implementation notes (for agents)

- **Fonts:** `app/layout.tsx` loads **Libre Baskerville** (`--font-serif`) and **Source Sans 3** (`--font-sans`). `globals.css` still mentions DM font names in comments/fallbacks; treat the layout variables as source of truth.
- **Stack:** Next.js 14 App Router, CSS Modules + global CSS (no Tailwind in this repo).
